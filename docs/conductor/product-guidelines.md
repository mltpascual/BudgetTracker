# Tipid — Product Guidelines

## Brand Voice and Tone

Tipid's voice is **friendly, warm, and approachable** — like a helpful friend who happens to be good with money. The tone avoids financial jargon and instead uses simple, everyday language. The carabao mascot reinforces this personality: calm, reliable, and endearing.

| Attribute | Do | Don't |
|-----------|-----|-------|
| **Friendly** | "Set a monthly budget para hindi ma-overspend." | "Configure your budget allocation parameters." |
| **Encouraging** | "You're 36% closer to your goal!" | "Warning: Goal not yet met." |
| **Simple** | "Track your spending" | "Monitor your expenditure patterns" |
| **Filipino-flavored** | Mix Taglish naturally when in Filipino mode | Use overly formal Filipino |

## Terminology and Glossary

| Term | Meaning | Notes |
|------|---------|-------|
| **Wallet** | A financial account (Cash, Bank, E-Wallet, Credit) | Not "Account" to avoid confusion with user accounts |
| **Quick Add** | Pre-saved transaction templates for one-tap logging | e.g., "Grab to Work ₱150.00" |
| **Goal** | A savings target with a deadline | e.g., "Emergency Fund — ₱100,000 by Dec 2026" |
| **Debt** | Money owed (either direction) | "owe" = I owe someone, "owed" = someone owes me |
| **Kalabaw/Carabao** | The app mascot — a cute brown water buffalo | Appears on Dashboard, Landing page, and tips |

## Error Message Conventions

- Use friendly, non-technical language
- Provide a clear action the user can take
- Example: "Oops! Something went wrong. Try refreshing the page."

## User-Facing Copy Standards

- Use **Sentence Case Per Word** for button labels and headings (e.g., "Add Transaction", "Monthly Budget")
- Currency amounts always include the symbol prefix: `₱17,461.00`
- Dates use readable format: "Mar 19, 2026"
- Percentages are shown as integers when possible: "36%"

## Documentation Style

- Technical docs use Markdown
- Code examples use TypeScript
- File and component names use PascalCase for React components, camelCase for utilities
